# 页面内查找（Find in Page）— 交互与实现方案

> 目标：为 MeoBrowser 提供轻量、键盘可达的「在当前网页中查找」能力：工具栏按钮 + ⌘F 弹出浮动查找条，高亮全部匹配，支持上一处 / 下一处循环跳转；默认字面匹配（忽略大小写），并可一键切换到 `*` 通配符模式。  
> 状态：**FI-0～FI-2 已实现**（2026-07-20）  
> 开发计划：[find-in-page-development-plan.md](find-in-page-development-plan.md)  
> 关联：[design.md](design.md) · [professional-features-roadmap.md](professional-features-roadmap.md) · [download-design.md](download-design.md)（浮层锚点范式） · [docs/sbkit/text-input.md](../sbkit/text-input.md)

## 1. 方案定位

### 1.1 做什么（V1）

| 能力 | 说明 |
|------|------|
| 工具栏入口 | 地址栏右侧 ActionGroup 增加「查找」按钮（SF Symbol `magnifyingglass`） |
| 快捷键 | ⌘F 打开 / 聚焦查找条；Esc 关闭；F3 / ⌘G 下一处；⇧F3 / ⌘⇧G 上一处 |
| 浮动查找条 | 关键词输入、清空、上一处 / 下一处、匹配计数、模式标识 |
| 高亮全部 | 页内所有匹配同时高亮；当前命中用更强对比色 |
| 默认匹配 | **字面模式** + **忽略大小写** |
| 通配符模式 | 前缀模式开关切换；`*` 匹配任意长度子串 |
| 循环跳转 | 到末尾后再跳回开头（wrap）；到开头后再跳到末尾 |
| 每标签状态 | 查询词、模式、当前索引随标签保留；切标签互不串扰 |

### 1.2 不做什么（V1）

- 不做全站 / 跨标签统一搜索（那是历史或命令面板的事）
- 不做完整正则（`()`、`|`、量词等）；通配符仅 `*`（可选 V1.1 加 `?` 单字符）
- 不替换文案（Find & Replace）
- 不搜索 PDF / 插件文档视图（仅 HTML 文档树）
- 不劫持页面内已有的站点自建搜索 UI
- 不做「在网页中搜索引擎再搜一遍」（地址栏已有）

### 1.3 设计原则

1. **Chrome 同构**：浮层语言对齐下载面板 / 地址栏补全（锚点、Esc、点外部关闭）。  
2. **键盘优先**：专业用户主要靠 ⌘F / F3 / ⌘G，鼠标是补充。  
3. **所见即所得**：默认高亮全部匹配；计数始终可见，消除「有没有找到」的不确定感。  
4. **两种模式显式切换**：字面与通配符不靠猜（见 §3），避免把字面 `*` 误当成通配。  
5. **输入控件走 SBKit**：关键词框用 `SBTextField`，并保证编辑菜单快捷键可用。

## 2. 用户场景

### 2.1 典型流程

```
阅读长文 / API 文档
  → ⌘F 或点工具栏 🔍
  → 查找条弹出并聚焦；若有选区则自动填入
  → 输入 "webkit"
  → 页面高亮全部；计数显示「3 / 12」；视口滚到第 3 处（或首次为 1 / 12）
  → F3 / ⌘G 依次下跳；到 12 后再回 1
  → Esc 关闭查找条并清除高亮
```

### 2.2 通配符流程

```
想找「任意前缀 + Error」
  → 打开查找条 → 点左侧模式标识，切到「通配符」
  → 输入 "*Error"
  → 匹配 timeoutError、FatalError、Error 等
  → 再切回「字面」后，同一输入按字面 `*Error` 搜索（含星号字符）
```

### 2.3 适用与不适用

| 场景 | 行为 |
|------|------|
| 普通 HTML 网页 | ✅ 完整查找 |
| 新标签页 Launchpad | ❌ 不启用（无网页正文）；⌘F 可 no-op 或轻提示 |
| `about:` / 证书警告页 | ❌ 不启用 |
| iframe / shadow DOM 内文本 | ✅ V1 尽量覆盖同源可访问节点；跨域 iframe 尽力而为并在文档中注明限制 |
| 动态 SPA（DOM 变化） | ✅ 监听变更后防抖重扫（见 §5.4） |

## 3. 双模式如何并存（核心决策）

用户需求：「前面加一个标识后，就是支持通配符的模式」。需要同时支持**字面**与 **`*` 通配**，且默认字面、忽略大小写。

### 3.1 候选方案对比

| 方案 | 做法 | 优点 | 缺点 | 结论 |
|------|------|------|------|------|
| **A. 前缀模式开关（推荐）** | 输入框左侧一枚可点标识：字面 ⇄ 通配符 | 发现性好；两种语义绝不混淆；符合「前面加标识」 | 多一次点击（可记住上次模式） | **V1 采用** |
| B. 自动含 `*` 即通配 | 查询里出现 `*` 就进通配 | 少一次操作 | 搜代码里的字面 `*`、脚注 `*` 会翻车 | 不采用 |
| C. 语法前缀（如 `*:foo*`） | 以 `*:` 开头表示通配 | 键盘党友好 | 学习成本高；与真实前缀冲突 | 不采用作主路径 |
| D. 直接上正则 | `/foo.*bar/i` | 能力最强 | 易误伤、难解释、转义地狱 | **V2 可选** |

### 3.2 定稿：前缀模式开关 + 会话记忆

```
┌──────────────────────────────────────────────────────────────┐
│ [ Aa▾ ] [ 🔍|✱ ] [ 关键词…………… ✕ ]  [ ˄ ] [ ˅ ]   3 / 12   │
│   ↑         ↑         ↑ 清空          ↑上一 下一    计数     │
│ 选项菜单   模式标识                                             │
└──────────────────────────────────────────────────────────────┘
```

| 控件 | 行为 |
|------|------|
| **模式标识**（输入框前） | 两态切换。**字面**：`magnifyingglass` 或无角标；**通配符**：图标叠加 `✱` / 小号 `*`，tooltip「通配符：* 匹配任意字符」。切换后立即按新语义重搜 |
| **选项菜单**（可选，Aa） | V1 至少含「区分大小写」；默认关（忽略大小写）。为 V2「全字匹配 / 正则」预留入口 |
| **会话记忆** | 同一窗口内记住「上次模式 + 是否区分大小写」；新窗口用默认（字面 + 忽略大小写） |

### 3.3 通配符语义（V1）

| 模式 | 查询示例 | 含义 |
|------|----------|------|
| 字面 | `foo*bar` | 连续字符 `f o o * b a r`（星号是普通字符） |
| 通配符 | `foo*bar` | `foo` + 任意长度（含空）+ `bar` |
| 通配符 | `*Error` | 以 `Error` 结尾的可见文本片段 |
| 通配符 | `*` | 特殊：视为无效查询，不扫全页（避免整页高亮爆炸） |
| 通配符 | 空 | 清除高亮，计数 `0` |

实现上：通配模式把用户输入转成「安全正则」——仅把 `*` → `[\s\S]*?`，其余元字符全部转义。**用户看到的是通配符，不是正则。**

> **创意预留（V1.1）**：通配模式下支持 `?` = 恰好一个字符；选项菜单增加「显示通配符帮助」一行 tip。

### 3.4 为什么不默认「看见星号就通配」

页面内查找常用于**文档、代码、错误信息**。字面 `*`、`*.js`、Markdown 强调残留等都很常见。显式开关用一次点击换来零歧义，比「智能猜测」更符合专业浏览器气质。

## 4. 交互设计

### 4.1 Chrome 落点

```
[ ← → ↻ ] [ ========== 地址栏 ★ ========== ] [ 🔍 ↓ ⋯ ]
                                              ↑
                                         查找（ActionGroup）
```

- ActionGroup `itemID`: `findInPage`  
- SF Symbol: `magnifyingglass`  
- Tooltip:「在页面中查找」  
- 可拖拽排序 / 溢出进「更多」，与下载、登录助手同机制  
- **建议默认顺序**：紧挨下载之前或之后（高频浏览工具）

### 4.2 查找条形态与锚点

| 项 | 定稿 |
|----|------|
| 容器 | 非模态浮层（优先 `NSPanel` 或 content 顶部附属条）；需能成为 key，以便输入 |
| 锚点 | **首选**：贴在网页内容区**右上角内侧**（Chrome / Safari 习惯，不挡地址栏操作） |
| 备选 | 锚在 🔍 按钮下方（对齐下载面板）；窄窗时自动改贴内容区顶栏 |
| 宽度 | 约 360–420 pt，随窗口变窄可压缩到约 300 pt |
| 层级 | 高于 WebView，低于模态 sheet；不挡住标签拖拽 |

**创意：吸附与收起**

- 打开时带短位移 + 淡入（约 120–160 ms），关闭反向。  
- 连续 F3 跳转时，若条会挡住当前命中，可短暂「半透明 + 上移」或自动避让命中矩形（V1.1）。  
- **精简态（可选 V1.1）**：失焦且仍有查询时，收成右上角小胶囊 `3/12`，点一下展开。

### 4.3 查找条控件细则

| 区域 | 说明 |
|------|------|
| 模式标识 | 见 §3.2；键盘：⌃⌘F 或选项菜单切换（V1 至少支持鼠标） |
| 关键词 | `SBTextField`；placeholder「查找」；输入即搜（**防抖 80–120 ms**） |
| 清空 ✕ | **仅当有文字时**显示在字段内右侧（`trailingView` / inset）；清空后焦点仍在字段 |
| 上一处 / 下一处 | SF Symbol `chevron.up` / `chevron.down`；无匹配时 disabled |
| 计数 | 格式 **`当前 / 总数`**（如 `3 / 12`）；无匹配 `0 / 0` 或 `未找到`；搜索中可短暂 `…` |
| 关闭 | 条上可不放 ✕（Esc / 再按 ⌘F 聚焦已开条更常见）；若放，用小号 `xmark` |

**计数创意**：总数很大（如 > 999）显示 `3 / 999+`，避免布局抖动。

### 4.4 打开 / 关闭 / 聚焦

| 事件 | 行为 |
|------|------|
| ⌘F 或点 🔍 | 若条未开 → 打开并聚焦；若已开 → **聚焦并全选**查询词（方便改搜） |
| 打开时有选区 | 将选区纯文本填入（截断过长，如 200 字）；立即搜索 |
| Esc | 关闭条 + **清除页内高亮** + 焦点回 WebView |
| 点条外网页区域 | 条可保持打开（方便边看边搜）；高亮保留。**点 chrome 空白不强制关** |
| 导航完成 / 换页 | 清除高亮；**保留查询词与模式**（方便新文档继续搜同一词） |
| 关标签 | 丢弃该标签查找状态 |
| 再按 🔍 | 与 ⌘F 相同（toggle 聚焦，不强制关闭） |

> **定稿**：关闭用 Esc；⌘F 不负责「关掉」，只负责「打开并聚焦」。若需快捷关闭，可后续加 ⌘⇧F 或菜单「隐藏查找」。

### 4.5 键盘映射（定稿）

| 快捷键 | 动作 | 备注 |
|--------|------|------|
| ⌘F | 打开 / 聚焦查找条 | 编辑菜单「在页面中查找」 |
| Esc | 关闭并清高亮 | 条为第一响应者时 |
| Return / F3 / ⌘G | 下一处 | 条内 Return = 下一处 |
| ⇧Return / ⇧F3 / ⌘⇧G | 上一处 | |
| ⌘E | 将选区填入查找条（条可未开则先开） | macOS 传统「使用所选内容来查找」；建议 V1 做 |
| ⌘L | **不占用** | 留给地址栏聚焦（路线图） |

菜单落点：`编辑 → 在页面中查找` ⌘F；`查找下一个` / `查找上一个`（可放在同一菜单或「查找」子菜单）。

### 4.6 页内高亮视觉

| 状态 | 建议样式 |
|------|----------|
| 非当前匹配 | 浅黄 / 系统 `findHighlightColor` 类浅底 + 圆角 2pt |
| 当前匹配 | 更深橙或强调色底 + 可选细描边；保证对比度 |
| 深色网页 | 注入样式用半透明高对比色，避免「白底黄字」在暗页上刺眼 |
| 滚动 | 跳转时 `scrollIntoView({ block: 'center' })`；减少命中贴边 |

**创意：命中涟漪（可选）**  
当前命中切换时，短促 scale/opacity 脉冲一次（150 ms），帮助眼动定位——尤其在密集匹配时比纯变色更明显。

### 4.7 边界态文案

| 状态 | UI |
|------|------|
| 空查询 | 无高亮；计数隐藏或 `—` |
| 有查询无匹配 | 输入框轻抖或计数用次要色 `0 / 0`；不弹系统 alert |
| 通配符无效（仅 `*`） | 计数旁小号 tip「请输入更多字符」 |
| 页面不可查 | 打开条时 placeholder「当前页无法查找」且输入 disabled |

## 5. 技术架构

### 5.1 为何不用纯 `WKWebView findString:`

macOS 11+ 提供：

```objc
[webView findString:query
  withConfiguration:config
  completionHandler:^(WKFindResult *result) { ... }];
```

`WKFindConfiguration` 支持 `backwards` / `caseSensitive` / `wraps`，适合「跳到下一处」。

但 V1 硬需求是：

1. **同时高亮全部匹配**  
2. **显示 `当前 / 总数`**  
3. **通配符模式**

原生 API **不提供** match count、highlight-all、通配/正则。因此：

| 路径 | 用途 |
|------|------|
| **主路径：JS 注入引擎** | 收集文本节点、匹配、包 `<mark>`（或自定义 span）、计数、跳转、模式语义 |
| **可选辅路径：`findString:`** | 仅作退化或对照；V1 可不依赖，避免两套高亮打架 |

项目已有 `WKUserScript` + `evaluateJavaScript:` + message handler 基础设施（登录助手、验证码、Feed），查找引擎同构接入成本低。

### 5.2 模块划分

```
BrowserWindowController / BrowserMenus
        │ ⌘F、菜单、ActionGroup 按钮
        ▼
BrowserFindBarController          ← 每窗口一个；协调 UI ↔ 当前标签
        │
        ├─ BrowserFindBarView     ← 浮动条 UI（SBTextField、按钮、计数）
        │
        └─ 当前 BrowserTab 的 BrowserFindSession
                │
                ▼
        BrowserFindEngine (JS)    ← 注入 + evaluate；返回 JSON 结果
                │
                ▼
            WKWebView 文档
```

| 类型 | 职责 |
|------|------|
| `BrowserFindBarView` | 纯 UI：模式标识、输入、清空、上下、计数 |
| `BrowserFindBarController` | 开关条、锚点布局、快捷键、与窗口/标签同步 |
| `BrowserFindSession` | 每标签：query、mode、caseSensitive、currentIndex、matchCount |
| `BrowserFindEngine` | 组装 JS、执行 search/next/prev/clear、解析结果 |
| ActionGroup `findInPage` | 工具栏入口 |

建议目录：`SimpleBrowser/FindInPage/`。

### 5.3 引擎协议（概念）

```js
// 伪 API：注入到页面（isolated world 或 main，择一；优先与现有脚本策略一致）
MeoFind.search({
  query: "foo*bar",
  mode: "literal" | "wildcard",
  caseSensitive: false
}) → { matchCount: 12, currentIndex: 1 }  // 1-based；0 表示无

MeoFind.next()  → { currentIndex, matchCount, wrapped: true/false }
MeoFind.prev()  → { ... }
MeoFind.clear()
MeoFind.setCurrent(index)
```

原生侧收到结果后更新计数标签；`wrapped == YES` 时可在计数旁闪一下「已从头开始」类提示（创意，低打扰）。

### 5.4 扫描与安全约束

1. **只扫可见文本节点**（跳过 `script` / `style` / `noscript` / `textarea` 默认值策略可讨论；V1 建议跳过 `script/style`，保留普通文本与可阅读元素）。  
2. **高亮包裹**不破坏输入控件：不进入 `input` / `textarea` / `contenteditable` 内部（或只读选中不包 mark）。  
3. **MutationObserver** 防抖重搜（约 200–300 ms），应对 SPA；重搜尽量保持「同一逻辑下的当前索引」。  
4. **性能**：匹配上限例如 **2000**；超出则高亮前 N 个，计数显示 `当前 / 2000+`，并停止继续包 DOM。  
5. **XSS**：查询只作文本/转义后正则源，**禁止**把原始字符串拼进 `innerHTML`。  
6. **导航**：`didCommit` / `didFinish` 时 `clear` 高亮；session 的 query 可保留。

### 5.5 与现有系统协作

| 现有能力 | 关系 |
|----------|------|
| `BrowserAddressBarActionGroup` | 注册 `findInPage` 按钮；暴露 `findInPageButton` 供锚点 |
| `BrowserMenus` / `SBApplicationMenus` | 增加查找菜单项；勿与编辑菜单的拷贝粘贴冲突 |
| `SBTextField` | 查找输入必须使用；配置清空按钮 inset |
| 多标签 / 多窗口 | Session 跟标签；Controller 跟窗口 |
| 下载面板等浮层 | z-order 与互斥：可同时存在；查找条优先接收 ⌘G |
| 地址栏补全面板 | 打开查找时不强制关补全；焦点在查找条时补全应已关闭 |

## 7. 分阶段交付

| 阶段 | 名称 | 范围 |
|------|------|------|
| **FI-0** | 骨架 | ActionGroup 按钮、查找条 UI、⌘F / Esc、SBTextField、无引擎占位 |
| **FI-1** | 字面查找 | JS 引擎：忽略大小写、高亮全部、计数、上下跳转、wrap、F3/⌘G |
| **FI-2** | 通配与打磨 | 模式标识、通配语义、选区填入、⌘E、导航清理、Mutation 防抖、上限 |

## 10. 开放问题（实现前可再拍板）

1. **查找条锚点**：内容区右上 vs 按钮下方——建议默认内容区右上，窄屏再回退按钮锚点。  
2. **是否搜索 `input` 的 value**：V1 建议否（避免打断表单）；专业用户若需要可进 V2。  
3. **⌘F 在 Launchpad**：静默忽略 vs 简短 HUD「没有可查找的页面」——建议静默忽略，少打扰。  
4. **区分大小写入口**：独立 `Aa` 按钮 vs 塞进选项菜单——按钮更显眼，菜单更省宽；窄条优先菜单。

## 11. 总结

MeoBrowser 的页面内查找采用 **工具栏 🔍 + ⌘F 浮动条**，以 **JS 高亮引擎**满足「全部高亮 + 计数 + 通配符」，用 **输入框前的模式标识** 干净地同时支持字面与 `*` 通配，避免自动猜星号。默认忽略大小写、F3/⌘G 循环跳转、每标签会话隔离，并与现有 ActionGroup / SBKit / 浮层范式对齐。

首版（FI-0～FI-2）即可达到 Safari/Chrome 主力路径的可用性；涟漪、胶囊收起、正则等作为明显但不阻塞上线的增强项。
