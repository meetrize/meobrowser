# MeoBrowser — 面向技术/专业用户的功能规划

> 目标读者：产品决策、后续迭代开发  
> 产品定位：macOS 原生、轻量、**为技术人员与专业人士优化工作流**的浏览器  
> 关联文档：[design.md](design.md) · [multi-tab-design.md](multi-tab-design.md) · [new-tab-launchpad-design.md](new-tab-launchpad-design.md)

---

## 1. 产品定位

### 1.1 一句话

MeoBrowser 不是 Chrome 的替代品，而是 **「日常浏览 + 专业工作流入口」**：用系统 WebKit、原生 AppKit 保持轻量，在键盘效率、开发调试、信息检索上为技术/专业用户做深。

### 1.2 目标用户

| 画像 | 典型场景 | 核心诉求 |
|------|----------|----------|
| **软件开发者** | 查文档、调试本地服务、Review PR、运维面板 | 快速打开 localhost、调 Web Inspector、证书/代理可控 |
| **运维 / SRE** | Grafana、Kibana、云控制台、工单系统 | 多环境区分、会话恢复、标签分组、稳定长会话 |
| **数据 / 分析** | Notebook、BI、内部报表、API 文档 | 固定工作区、快捷入口、少干扰 |
| **技术写作者** | 官方文档、Stack Overflow、GitHub | 阅读体验、复制链接/Markdown、分屏对照 |
| **专业从业者（非纯开发）** | 法律检索、医学文献、设计规范站 | 可预测的界面、可靠的历史/书签、专注模式 |

### 1.3 设计原则

1. **键盘优先**：常用操作应有快捷键；复杂操作用命令面板统一入口。
2. **原生优先**：复用 macOS WebKit、系统 Web Inspector、系统代理与钥匙串，不自造轮子。
3. **轻量可预期**：功能可以少，但行为必须稳定、可解释；不做隐藏的数据采集。
4. **工作流而非功能堆砌**：每个功能应回答「专业用户省下了哪一步」。
5. **与 Launchpad 协同**：新标签页快捷方式 + 地址栏补全是「个人工作区」核心，扩展时保持单一数据源。

### 1.4 明确不做（保持边界）

- 不自研渲染引擎或 Chromium 级扩展生态
- 不做完整密码管理器、邮件客户端、笔记应用
- 不追求「所有网站完美兼容」——优先保证文档站、Git 平台、控制台、本地 dev 体验
- 不做跨平台（专注 macOS 体验深度）

## 2. 现状基线（2026-07）

以下能力**已实现**，后续规划应在其上迭代，避免重复建设。

| 模块 | 能力 |
|------|------|

| 设置 | 默认搜索引擎选择 |
| 基础架构 | AppKit + WKWebView、SBKit 文本输入、Makefile 构建 |

## 3. 功能建议总览

按 **P0（强烈建议）→ P1（高价值）→ P2（增强）→ P3（可选）** 排列。

### 3.1 导航与键盘效率 — P0

专业用户的时间大多花在「找页面 → 切换页面 → 输入 URL/搜索」上。

| 功能 | 说明 | 理由 |
|------|------|------|
| **命令面板（Command Palette）** | ⌘⇧P 弹出模糊搜索：打开 URL、切换标签、执行浏览器命令 | 统一入口，学习成本低，可渐进扩展 |
| **标签快速切换** | ⌘1～9 切换前 N 个标签；⌘⇧T 恢复刚关闭标签 | 与 Chrome/Safari  muscle memory 一致 |
| **聚焦地址栏** | ⌘L / ⌘K 聚焦并全选 | 文档查阅时高频操作 |
| **停止加载** | Esc 或工具栏停止按钮 | 慢文档/CDN 故障时必备 |
| **加载进度** | 顶栏细进度条（`estimatedProgress`） | 长页面加载反馈 |
| **固定标签（Pin Tab）** | 标签右键/快捷键固定，图标化、不可误关 | 文档站、控制台、Git 常驻 |

**实现提示**：命令面板可先做 20～30 条内置命令 + 快捷方式/打开 URL，无需一开始接历史记录。

### 3.2 开发者工具与调试 — P0

这是与「普通轻量浏览器」最大的差异化。

| 功能 | 说明 | 理由 |
|------|------|------|
| **打开 Web Inspector** | 菜单 + ⌘⌥I，调用 `WKWebView` 的检查器 API | 调试前端、看 Network/Console 的基础 |
| **查看网页源代码** | 右键或命令面板 | 查文档结构、排查 SSR 问题 |
| **localhost 友好** | `localhost`、`127.0.0.1`、`*.local` 默认可访问；可选「开发模式」 | 本地 dev 是开发者第一场景 |
| **自签名证书处理** | 开发模式下允许继续访问 + 明确警告（非生产默认） | 内网/HTTPS dev 常见痛点 |
| **自定义 User-Agent** | 设置中预设：默认 / Mobile / 自定义字符串 | 响应式调试、爬虫规则验证 |
| **禁用缓存（硬刷新）** | ⌘⇧R 绕过缓存 reload | 与前端工作流一致 |

**实现提示**：Web Inspector 在 macOS 上可通过 `[_WKInspector` 或 Safari 开发菜单模式；优先查当前 WebKit 公开 API，避免私有 API 依赖。

### 3.3 工作区与 Launchpad 深化 — P0 / P1

现有 Launchpad 已是「个人仪表盘」，应向「专业工作区」演进。

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **快捷方式文件夹/分组** | P0 | 如「工作 / 个人 / 运维」，Launchpad 分页或折叠 |
| **环境标签（Staging / Prod）** | P1 | 同一服务不同 URL 成组，图标角标区分环境 |
| **Favicon 显示** | P1 | 标签栏 + Launchpad + 地址栏补全（multi-tab NTP-4 已规划） |
| **导入/导出快捷方式 JSON** | P1 | 团队共享模板、换机迁移 |
| **Launchpad 内搜索** | P1 | 快捷方式多时，输入过滤（仍可与地址栏补全共用逻辑） |
| **默认工作区布局** | P2 | 启动时恢复窗口大小、标签组、选中的 Launchpad 页 |

**原则**：快捷方式仍是单一数据源（`BrowserShortcutStore`），历史/书签若引入需设计合并策略，避免三套数据。

### 3.4 历史、书签与检索 — P1

专业用户需要「我上周看过的那份 RFC」。

| 功能 | 说明 |
|------|------|
| **浏览历史** | 按时间列出，支持标题/URL 搜索 |
| **历史补全** | 地址栏 AC-3：快捷方式 + 历史 + 最常访问 |
| **简单书签** | 与星标/Launchpad 打通：星标 = 快捷方式，或「书签仅收藏、Launchpad 仅常用」二选一 |
| **关闭标签前确认** | 可选：含未提交表单的标签提示（依赖 `WKWebView` 能力有限，可降级为「固定标签不可关」） |

**实现提示**：历史可用 SQLite 或轻量文件索引；注意隐私（可「退出时清除」选项）。

### 3.5 标签与窗口管理 — P1

| 功能 | 说明 |
|------|------|
| **标签拖拽排序** | multi-tab L2d 待办项 |
| **标签分组 / 颜色标记** | 如「文档」「工单」「监控」三色分组 |
| **多窗口** | 新窗口打开链接、窗口级会话恢复 |
| **分屏 / 侧边栏** | 左右两栏各一 `WKWebView`：文档 + 代码仓库对照 |
| **复制标签 URL / 复制为 Markdown 链接** | 写文档、Slack/飞书分享 |

### 3.6 网络、代理与安全 — P1

| 功能 | 说明 |
|------|------|
| **系统代理尊重 + 显式开关** | 企业网络、Charles/mitmproxy 调试 |
| **按站点清除 Cookie/存储** | 调试登录态、A/B 测试 |
| **第三方 Cookie 策略** | 设置项：默认 / 阻止跨站（文档站兼容需测试） |
| **HTTPS 仅模式（可选）** | 专业用户内网场景可关闭 |
| **无遥测 / 隐私说明** | 设置页声明：本地存储、无上报（建立信任） |

### 3.7 阅读与专注 — P2

| 功能 | 说明 |
|------|------|
| **阅读模式** | 提取正文、可调字体（技术文档长篇阅读） |
| **页面缩放** | ⌘+ / ⌘- / 重置，按站点记忆 |
| **最小 UI 模式** | 隐藏工具栏，仅保留标签栏（演示、截图） |
| **深色模式强制/跟随** | 对部分文档站 `color-scheme` 不友好的补偿 |

### 3.8 下载与文件 — P2

| 功能 | 说明 |
|------|------|
| **下载列表** | 显示进度、打开所在文件夹、取消（见 [download-design.md](download-design.md) V1） |
| **「链接另存为」** | V1：系统右键「下载链接的文件」，静默写入 Downloads |
| **拖拽下载项到 Finder / 终端** | V1 已支持拖到 Finder |

### 3.9 自动化与集成 — P2 / P3

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **AppleScript / Shortcuts 基础** | P2 | 打开 URL、新建标签——与 macOS 自动化生态衔接 |
| **URL Scheme** | P2 | `meobrowser://open?url=` 供 Alfred/Raycast 调用 |
| **复制为 cURL** | P3 | 从 Inspector 或简化版网络面板导出 |
| **每站点 User CSS/JS** | P3 | 轻量 userscript，改文档站样式、隐藏干扰元素 |
| **与终端联动** | P3 | 「在 iTerm 打开当前目录」类集成（偏 niche） |

### 3.10 质量与可观测 — 贯穿各阶段

| 能力 | 说明 |
|------|------|
| **崩溃/卡死恢复** | 重启后恢复标签，坏标签可单独关闭 |
| **内存可见性** | 可选：关于页显示标签数与内存（已有 `make stats-browser` 基础） |
| **可访问性** | VoiceOver、键盘全可达 |
| **国际化** | UI 中文为主，代码与 URL 处理保持 Unicode 正确 |

## 4. 推荐路线图

与现有 L1 → L2 → Launchpad → AC 的编号衔接，建议按 **里程碑** 推进。

### M1 — 专业基座（约 1～2 周）

**目标**：开发者「能拿 MeoBrowser 日常查文档 + 调本地页」。

- [ ] 命令面板（打开 URL、切换标签、基础命令）
- [ ] ⌘L / ⌘1～9 / ⌘⇧T / ⌘⇧R
- [ ] 加载进度 + 停止加载
- [ ] Web Inspector + 查看源代码
- [ ] 固定标签（Pin）

### M2 — 工作区（约 2 周）

**目标**：Launchpad 成为真正的个人/团队仪表盘。

- [ ] Launchpad 文件夹/分组
- [ ] Favicon（标签 + Launchpad + 补全）
- [ ] 快捷方式导入/导出
- [ ] 浏览历史 + 地址栏历史补全（AC-3）
- [ ] 标签拖拽排序（完成 multi-tab L2d）

### M3 — 多任务与网络（约 2～3 周）

**目标**：长时间专业会话、企业网络可用。

- [ ] 多窗口 + 窗口级会话
- [ ] 分屏（双 WebView）
- [ ] 代理与 Cookie 管理
- [x] 下载管理基础版（V1：见 [download-design.md](download-design.md)）
- [ ] 复制 Markdown 链接、环境分组

### M4 — 体验深化（持续）

- [ ] 阅读模式、缩放记忆
- [ ] AppleScript / URL Scheme
- [ ] 自签名证书开发模式
- [ ] 可选 User CSS/JS

## 5. 典型用户故事（验收参考）

### 5.1 本地开发

> 作为前端开发者，我在 `localhost:3000` 跑项目，需要一键硬刷新、打开 Inspector 看 Console，且自签名 HTTPS 不被 silently 挡掉。

**验收**：⌘⇧R 有效；⌘⌥I 打开检查器；开发模式下证书错误有「继续访问」且默认关闭。

### 5.2 文档 + 代码对照

> 我在写 SDK 集成，左侧 pinned React 文档，右侧 GitHub PR；需要 ⌘⇧] 快速切换且重启后恢复。

**验收**：分屏或双窗口；Pin + 会话恢复；标签分组可选。

### 5.3 运维值班

> 我固定打开 Grafana、PagerDuty、内部 Wiki，环境不同 URL 容易点错。

**验收**：Launchpad 分组 + 环境角标；命令面板输入 `graf` 跳到正确 staging。

### 5.4 写作与分享

> 我在技术博客引用了三个文档链接，需要从历史里找回上周的 RFC，并复制成 Markdown。

**验收**：历史搜索；复制 `[title](url)` 到剪贴板。

## 6. 与竞品差异（简要）

| 维度 | Chrome / Edge | Safari | MeoBrowser 方向 |
|------|---------------|--------|-----------------|
| 资源占用 | 高 | 低 | 低（单 WebKit 进程模型可控） |
| 扩展生态 | 极强 | 弱 | **不拼扩展**，做内置专业命令 |
| 开发调试 | DevTools 完整 | 需连 Mac Safari 开发菜单 | **内置 Inspector 入口** |
| 启动页 | 通用 | iCloud 标签 | **Launchpad 可定制工作区** |
| 隐私 | 账号体系强 | Apple 生态 | **本地优先、无账号** |

## 7. 开放问题（待产品拍板）

1. **书签 vs Launchpad**：合并为一套「快捷方式」，还是书签轻量、Launchpad 仅高频？
2. **配置文件/Profile**：是否需要「工作 / 个人」完全隔离（Cookie、历史、快捷方式）？
3. **搜索默认**：专业用户是否更偏好 DuckDuckGo / Kagi / 自建 SearXNG？
4. **分屏优先级**：是否先于「多窗口」做，以控制复杂度？
5. **团队能力**：导入/导出是否扩展到「团队共享 Launchpad 模板」（只读订阅）？
