# 标签栏自适应宽度 — 交互与实现方案

> 目标：多标签时窗口仍可任意缩放；标签栏跟随窗口收缩；放不下时用箭头菜单访问其余标签。  
> 关联：`multi-tab-design.md` §5.1；实现：`BrowserTabStripView`、`BrowserTabItemView`。

---

## 1. 原则

| # | 原则 |
|---|------|
| 1 | **窗口优先**：`window.minSize` 固定且偏小（当前 **400×300**），**绝不**随标签数量抬高 |
| 2 | **标签条适配**：先等宽伸缩；再不够则 **溢出箭头 + 菜单**（不用横向滚动） |
| 3 | **交通灯 / `+` / 溢出箭头固定**：仅中间标签区域参与伸缩 |
| 4 | **选中可见**：可见窗口始终包含当前选中标签 |

---

## 2. 交互流程

```
窗口变窄 / 标签变多
        ↓
① 在可用宽度内等宽平分（clamp 到 max～min）
        ↓
② 标题尾部截断（…）；关闭按钮随宽度/悬停显示
        ↓
③ 最小宽仍放不下全部：显示 ▾ 箭头，只铺可见标签
        ↓
④ 点击箭头 → 弹出菜单列出其余标签，点选即可切换
```

选中标签若不在当前可见段，布局会移动可见窗口，使其落在条内。

---

## 3. 数值规范

| 属性 | 值 | 说明 |
|------|-----|------|
| 标签最大宽 | **200 pt** | 少标签时不过分拉长 |
| 标签最小宽 | **108 pt** | 仍可辨认标题；低于此进入溢出 |
| 关闭按钮常显阈值 | **≥ 120 pt** | 再窄：选中常显，未选中悬停显示 |
| 标签间距 | 2 pt | 与现实现一致 |
| 溢出箭头宽 | **22 pt** | `chevron.down`，仅溢出时显示 |
| 窗口 `minSize` | **400×300** | 与标签数无关 |

宽度计算：

```
若 N×min + (N−1)×spacing ≤ 可用宽（不含箭头）：
  全部显示，等宽平分，隐藏箭头
否则：
  可用宽减去箭头占位
  visibleCount = max(1, floor((tabsWidth + spacing) / (min + spacing)))
  可见窗口包含 selectedIndex，其余进溢出菜单
```

---

## 4. UI 细节

| 项 | 行为 |
|----|------|
| 结构 | `[交通灯][可见标签…][▾?][+][trailing]` |
| 等宽 | 可见标签等宽；选中态用背景区分 |
| 溢出菜单 | 其余标签标题列表；勾选当前项（若在列表中）；点选 → 切换 |
| 关闭 | 宽 ≥ 120 或已选中：显示；窄且未选中：悬停显示 |
| 拖窗 | 空白区 / 非按钮区可拖窗、双击缩放 |

**不采用**：按标签数改 `minSize`；横向滚动作为主路径。

---

## 5. 实现要点

- `BrowserTabStripView`：裁剪容器 + frame 布局可见标签；`overflowButton` 宽度 0/22 切换  
- 隐藏的标签 `hidden = YES`，ID 记入 `overflowTabIDs`，供菜单使用  
- `BrowserTabItemView`：弹性宽、标题点击穿透到选中  

### 验收

- [ ] 多标签时窗口可拖至约 400 宽  
- [ ] 标签放不下时出现 ▾，点击菜单可切换到隐藏标签  
- [ ] 从菜单选中后，该标签出现在条上（可见窗口跟随）  
- [ ] 无溢出时箭头不占位
